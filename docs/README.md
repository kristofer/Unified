## Contributing to Documentation

Documentation is crucial for this project. When contributing:

1. **Keep docs up to date** - Update docs when changing features
2. **Be clear and concise** - Use simple language
3. **Add examples** - Show, don't just tell
4. **Check links** - Ensure all links work
5. **Use markdown** - Follow markdown best practices

### Documentation Standards

- Use Markdown format
- Include table of contents for long documents
- Add code examples where appropriate
- Use clear headings and structure
- Keep paragraphs short
- Use lists and tables for clarity
- Include diagrams when helpful
